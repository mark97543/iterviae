# Iter Viae - Version 1.2 "Primus"
**Release Date:** April 30, 2026
**Theme:** Dynamic Planning & Organization

Version 1.2 introduces the first major interactive planning feature: Draggable Stop Reordering. This update allows users to visually organize their trip itinerary with real-time feedback.

---

## 🚀 Key Features

### 1. Draggable Itinerary Stops
The sidebar list has been upgraded from a static display to a fully interactive sorting interface.
- **DND-Kit Integration**: Leverages the high-performance `@dnd-kit` library for smooth, 60fps dragging.
- **Visual Feedback**: Added dedicated styling for the "dragging" state, including opacity shifts and depth (z-index) management.
- **Touch Support**: Implemented `touch-action: none` to ensure tablet and mobile users can sort stops without triggering browser scroll events.

### 2. Intelligent Re-Sorting Logic
Reordering items in a list is easy, but keeping them in order in a database is hard. 
- **Auto-Sort Engine**: Whenever a stop is dropped into a new position, the app automatically loops through the entire itinerary and recalculates the `sort` property for every stop (1-based indexing).
- **Directus Sync**: This ensures that when the trip is saved, the database receives the exact order seen on the screen.

---

## 🛡 Security & Safety

### 1. Edit Mode Guard
To prevent accidental reordering during a "view-only" session, the draggable handles are completely disabled unless the user has explicitly clicked the **Edit** button.
- **Logic**: The `useSortable` hook is dynamically disabled based on the global `editMode` state.

---

## 🛠 Stability Updates

### 1. Reactive Map Syncing
Because the map and route engines watch the global `stops` array, they now "react" instantly to drag-and-drop operations.
- **Recalculation**: Moving a stop immediately triggers a Valhalla route update, showing the new travel times and distances between the reordered legs.
