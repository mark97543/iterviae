# Iter Viae - Version 1.1 "Primus"
**Release Date:** April 30, 2026
**Theme:** Stability & Immersive Experience

Version 1.1 focused on maturing the application from a prototype into a "solid app" feel, specifically targeting mobile responsiveness and map data integrity.

---

## 🚀 Key Features

### 1. Map Auto-Focus (The "Glide")
The map now intelligently zooms and pans to your trip the moment it loads.
- **Dynamic Framing**: Automatically calculates a bounding box to fit your entire route on the screen with perfect padding.
- **Smart Fallback**: Detects if a trip is new (1 stop) and uses a smooth `flyTo` animation instead of a frame.
- **Collision Protection**: Implemented custom math to prevent crashes when multiple stops share the exact same coordinates.

---

## 🛠 Stability & Bug Fixes

### 1. The "Infinite Loop" Resolution
Fixed a critical bug in `Map.tsx` where the routing engine would re-trigger itself indefinitely, causing high CPU usage and browser lag. 
- **Solution**: Refined `useEffect` dependencies to ignore the `route` object itself during calculation.

### 2. Data Integrity & Persistence
Resolved multiple issues where trip data would corrupt or fail to save.
- **JSON Resilience**: Added robust parsing to handle cases where Directus returned `[object Object]` strings, allowing the app to clear and recalculate instead of crashing.
- **Manual Persistence**: Shifted from aggressive auto-saving (which caused lag) to a reliable, manual "Save Trip" workflow that prioritizes the active Trip ID.
- **Coordinate Sanitization**: Implemented forced numeric casting for all map points to prevent `TypeError` crashes caused by string-based coordinates.

### 3. Mobile Viewport Optimization
Transformed the web interface to feel like a native mobile application.
- **DVH Units**: Switched to Dynamic Viewport Height to prevent the Chrome address bar from "jumping" the layout.
- **Scroll Locking**: Disabled "rubbery" overscrolling and pull-to-refresh to keep the map and dashboard locked in place.
- **Safe Area Support**: Added support for phone notches and status bars using `viewport-fit=cover`.

---

## 📖 Engineering Lessons

### Why we switched to Manual Math for Bounds
We discovered that relying on the `LngLatBounds` object from a globally loaded library can be risky. If the library isn't fully ready, or if coordinates are passed as strings, the internal methods can fail silently. 
**Version 1.1** now uses raw JavaScript `Math.min/max` to calculate boundaries, making the map engine significantly more predictable.
